# OpenSpec-style workflows for SDO

OpenSpec is a useful inspiration for SDO's artifact lifecycle.

## Useful ideas

- Change folder per unit of work
- Separation of why, what, how, and steps
- Verify before archive
- Historical archive
- Lightweight and expanded workflows

## SDO adaptation

```text
OpenSpec: proposal → specs → design → tasks → apply → verify → archive
SDO:     intent → operational spec → plan → runbook → execute → validate → evidence/review/archive
```

## Warning

SDO must not become a software-change-only workflow. Operational targets include services, hosts, users, databases, networks, pipelines, APIs, and manual procedures.
